# Security Policy

## Reporting A Vulnerability

Please do not open a public issue with exploit details or sensitive information.

If GitHub private vulnerability reporting is enabled for this repository, use it. If it is not available yet, open a minimal public issue asking for a private security reporting channel and do not include technical exploit details.

Useful information for a private report:

- Affected version or commit.
- Safari and macOS versions.
- Clear reproduction steps.
- Impact and whether user data, page data, or extension permissions are involved.
- Any suggested mitigation.

## Supported Versions

Until the first public release, only the latest commit on `main` is supported.
