# ドキュメント言語ポリシー

この日本語版は `docs/process/documentation-language.md` の翻訳です。内容が異なる場合は英語版を優先します。

このプロジェクトでは、英語ドキュメントを正本とします。

日本語ドキュメントは、ユーザーやコントリビューター向けの翻訳として提供します。ただし、日本語版は英語版をもとに作成される翻訳であり、英語版と日本語版の内容が食い違う場合は英語版を正とします。

## 構成

- 正本となる英語ドキュメントは、リポジトリ直下または `docs/` 配下に置きます。
- 日本語翻訳は `docs/ja/` 配下に置きます。
- 日本語翻訳には、対応する英語版への参照を入れます。

## 更新フロー

ドキュメントを変更するときは、次の順番で進めます。

1. まず英語版を更新します。
2. ユーザー向け、またはコントリビューター向けの内容であれば、同じ Pull Request で日本語版も更新します。
3. 同じ Pull Request で日本語版を更新できない場合は、その旨を Pull Request に明記します。
4. 新しい挙動、コマンド、権限、運用ルールを日本語版だけに追加してはいけません。

## 翻訳スタイル

- 語順ではなく意味を翻訳します。
- コマンド名、ファイル名、キーボードショートカット、コード識別子は変更しません。
- ユーザー向け説明では自然な日本語を使います。
- 競合比較では、相手を否定するよりも「向いている用途の違い」を具体的に説明します。
