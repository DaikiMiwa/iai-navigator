# このアプリを選ぶ理由

この日本語版は `docs/strategy/why-use-this-app.md` の翻訳です。内容が異なる場合は英語版を優先します。

## SCQ-A

**Situation**:
Safari で普段のブラウジングをキーボード中心にしたいユーザーは、リンクを素早く選び、ページを少ないキー操作で移動したい。求めている中核体験は Vimium 風で、`f` でリンク Hint を表示し、表示された Hint を入力してリンクを開き、`h/j/k/l` と `gg` / `G` でページを移動すること。

**Complication**:
既存の選択肢は、ユーザーによっては過剰、重い、または保守継続性に不安がある。Vimlike は強力だが、コマンドや設定が多く、単純なリンク移動とページ移動だけを求めるユーザーには複雑に感じられることがある。Vifari は Hammerspoon と macOS のアクセシビリティ自動化を使うため、通常の Safari 拡張として使いたいユーザーには導入が重い。Vimari は近い選択肢だが、公開リリースとコード更新が古く、長期的な保守に不安を持つユーザーがいる。

**Question**:
Safari で Vimium 風の操作感を求めるユーザーは、なぜこのアプリを選ぶべきか。

**Answer**:
このアプリは、Safari で「リンクをキーボードで選ぶ」「ページをキーボードで移動する」という中核体験を、広範なブラウザ自動化スイートではなく、小さく理解しやすい Safari 拡張として使いたいユーザーに向いている。

## 未決事項

- **表現トーン**: ユーザー向けの文言では、競合を強く否定するよりも「向いている用途が違う」と説明する。
- **利用シーン**: 現時点の対象は、日常的な Web 閲覧、ドキュメント閲覧、調べ物。App Store 掲載文、業務利用、iPad 対応では別の位置づけが必要になる可能性がある。
- **証拠レベル**: 現時点の根拠は機能範囲と競合比較。Vimari の公開更新状況は GitHub の公開情報で確認済みだが、ユーザーインタビューや利用ログによる検証はまだない。

## ロジックツリー

**主張: Safari で最小限の Vimium 風操作を求めるユーザーは、このアプリを選ぶべきである。**

1. **必要な操作に絞っている**
   - `f` で現在 viewport に見えているリンクに Hint を表示できる。
   - Hint を最後まで入力すると、そのリンクの通常のクリック動作が現在タブで実行される。
   - `h/j/k/l` で小刻みにスクロールでき、`gg` / `G` でページ上端・下端へ移動できる。
   - タブ操作、履歴操作、検索、カスタムスクリプト、詳細設定は MVP に含めない。

2. **多機能拡張より理解しやすい**
   - Vimlike は強力だが、コマンドや設定が多く、リンク移動とページ移動だけをしたいユーザーには複雑に感じられることがある。
   - このアプリは「リンクを選ぶ」「ページを移動する」に用途を限定するため、覚えるキーが少ない。
   - 機能が少ない分、動作の期待値が明確になり、日常利用での驚きが減る。

3. **Hammerspoon のような外部自動化を必要としない**
   - Vifari は Hammerspoon と macOS アクセシビリティ自動化を使うため、導入の心理的・技術的ハードルが上がる。
   - このアプリは Safari Web Extension として動くため、ユーザーは「Safari に読み込む拡張」として理解できる。
   - システム全体を操作するのではなく、Safari 内のページ操作に範囲を絞る。

4. **信頼しやすく、保守しやすい**
   - Vimari は近い選択肢だが、2026-05-24 時点で最新 GitHub release は 2020-09-10 の `v2.1.0`、default branch の最新 commit は 2021-04-12 である。
   - Safari は macOS やブラウザ更新の影響を受けるため、最近の活動が見えにくい拡張には互換性や保守への不安が残る。
   - このアプリは、現行 Safari 向けに作られ、更新状況が見える小さな拡張を目指す。
   - MVP には telemetry、settings sync、custom script 実行、広範なブラウザ自動化を含めない。
   - 主な処理は、見えているリンクの検出、Hint 表示、クリック発火、ページ移動に限られる。
   - 小さいスコープは、ユーザーにとってもコントリビューターにとっても挙動を確認しやすい。

## 論理チェック

- **Q/A 対応**: Q は「なぜユーザーが選ぶべきか」。A は「中核操作に絞った軽い Safari 拡張だから」と答えている。
- **So what?**: Vimlike が広すぎる、Hammerspoon ベースの自動化が重い、古い拡張の継続性が気になる、というユーザーに焦点の合った選択肢を提供できる。
- **Why so?**: 機能範囲が小さく、導入モデルが Safari 拡張で、操作対象もリンク選択とページ移動に限定され、かつ保守状況を見える形にできるため。
- **MECE 確認**: 第一階層は「機能適合」「理解しやすさ」「導入の軽さ」「信頼性と保守性」。一部重なりはあるが、ユーザー価値としては別の観点になっている。

## 表現メモ

- 「このアプリを使うべき」は押し付けが強いので、ユーザー向けには「このアプリが向いているユーザー」「このアプリを選ぶ理由」が自然。
- 「Vimlike は複雑すぎる」という一般的な断定は避ける。「Vimlike は強力だが、単純なリンク移動だけを求めるユーザーには複雑に感じられることがある」と書く。
- 「Hammerspoon はやりすぎ」は、説明文では「外部自動化レイヤーを導入したくないユーザーもいる」と言い換える。
- 「Vimari は古い」は文脈なしに断定しない。「公開リリースとコード更新が古く、継続性に不安がある」と書く。

## Executive Summary

このアプリは、Safari で Vimium のようなキーボード操作を使いたいが、多機能な拡張や Hammerspoon のような外部自動化までは必要ないユーザーに向いている。MVP では、`f` によるリンク Hint、Hint 入力による現在タブでのリンク実行、`h/j/k/l` のスクロール、`gg` / `G` の上下移動に機能を絞る。

このアプリを選ぶ理由は、機能の多さではなく、機能を絞っていることにある。Vimlike は強力だが、単純なリンク移動とページ移動だけを求めるユーザーには複雑に感じられることがある。Vifari も強力だが、Hammerspoon ベースの構成は、Safari 拡張として気軽に使いたいユーザーには導入が重い。Vimari は近い選択肢だが、公開リリースとコード更新が古く、継続性に不安を持つユーザーもいる。このプロジェクトは、その間にある「小さく、分かりやすく、確認しやすく、現行 Safari 向けに保守される Safari 拡張」という選択肢を目指す。

したがって、このアプリの最も自然な位置づけは「Safari でリンク選択とページ移動だけをキーボード化したいユーザー向けの、軽量なナビゲーション拡張」である。今後は、タブ操作や詳細設定を急いで追加するより、まずはリンク Hint とページ移動の安定性、分かりやすさ、日常利用での違和感の少なさを優先すべきである。
